# 🧩 Vol 4. UX/UI 컴포넌트 대사전 (UI Components A-Z)

실무에서 "그 팝업창 띄우고", "밑에 동동 떠다니는 버튼 누르면", "옆으로 슉 넘기는 거" 라고 부르는 추상적인 명칭들을 글로벌 표준 UI 프레임워크(Material Design, Apple HIG, Bootstrap, Tailwind) 기준으로 총망라한 마스터 사전입니다.

---

## A
*   **Accordion (아코디언)**
    *   *설명*: 클릭하면 본문이 아래로 확장(Expand)되며, 다른 항목을 클릭하면 기존 항목이 닫히는(Collapse) 목록 형태. 주름상자 악기에서 유래. 주로 FAQ(자주 묻는 질문)에 쓰임.
*   **Alert (얼럿 / 경고창)**
    *   *설명*: 사용자의 즉각적인 행동이나 인지를 요구하는 중요 메시지. 치명적인 오류나 파괴적 행동(계정 삭제 등) 전에 띄움.
*   **App Bar (앱 바) / Header / Navbar**
    *   *설명*: 화면 최상단(또는 하단)에 위치하여 뒤로 가기, 현재 페이지 제목, 검색/설정 아이콘 등을 담고 있는 막대.
*   **Avatar (아바타 / 프로필)**
    *   *설명*: 사용자의 프로필 사진이나 이니셜을 담는 원형(또는 둥근 사각형) 컴포넌트. 상태 표시(온라인/오프라인 초록색 점)를 동반하기도 함.

## B
*   **Badge (배지)**
    *   *설명*: 아바타나 아이콘의 우측 상단에 붙어 알림 갯수(`N`)나 새로운 소식(`N`)을 빨간색 원으로 알려주는 작은 컴포넌트.
*   **Bottom Navigation (바텀 내비 / 하단 탭)**
    *   *설명*: 모바일 앱 하단에 위치한 3~5개의 핵심 메뉴 탭.
*   **Bottom Sheet (바텀 시트)**
    *   *설명*: 모바일에서 액션 유도를 위해 화면 아래에서 위로 스윽 올라오는 패널. 공유하기 창이나 필터 옵션 선택 창으로 자주 쓰임.
*   **Breadcrumbs (브레드크럼 / 빵부스러기)**
    *   *설명*: 웹사이트 상단에 `Home > Men's Clothing > Shoes > Sneakers` 처럼 현재 사용자의 계층적 위치를 보여주는 내비게이션 경로.
*   **Button (버튼)**
    *   *세부 종류*:
        *   `Contained / Solid Button`: 속이 색상으로 꽉 찬 가장 중요한 CTA 버튼.
        *   `Outlined / Ghost Button`: 테두리만 있고 속이 투명한 버튼. 차순위 액션에 쓰임.
        *   `Text / Flat Button`: 테두리나 배경 없이 텍스트만 있는 버튼. 덜 중요한 동작(예: '취소')에 쓰임.

## C
*   **Card (카드)**
    *   *설명*: 이미지, 제목, 본문, 액션 버튼 등을 하나의 덩어리로 묶어 놓은 둥근 직사각형 컨테이너.
*   **Carousel (캐러셀 / 슬라이더)**
    *   *설명*: 여러 장의 이미지나 카드를 좌우(가로)로 스와이프하여 넘겨볼 수 있는 컴포넌트. (회전목마에서 유래).
*   **Checkbox (체크박스)**
    *   *설명*: 여러 개의 옵션 중 **'다중 선택'**이 가능할 때 사용하는 네모난 체크 박스.
*   **Chip / Tag / Pill (칩 / 태그 / 필)**
    *   *설명*: 텍스트를 감싸고 있는 알약(Pill) 모양의 작은 덩어리. 검색어 필터링, 카테고리 표시, 연락처 이름 등에 쓰임.

## D
*   **Dialog (다이얼로그) / Modal (모달)**
    *   *설명*: 기존 화면 위에 겹쳐 뜨는 창. **반드시 사용자 입력이나 닫기를 수행해야만 원래 화면으로 돌아갈 수 있는(Blocking) 강제성**을 띰.
*   **Divider (디바이더 / 구분선)**
    *   *설명*: 콘텐츠 그룹을 시각적으로 분리하기 위해 긋는 1px의 얇은 가로/세로 선.
*   **Drawer (드로어 / 사이드 내비게이션)**
    *   *설명*: 화면 밖(주로 좌측)에 숨어 있다가, 햄버거 메뉴(☰)를 누르면 화면을 덮으며 서랍처럼 밀려 나오는 패널.
*   **Dropdown (드롭다운)**
    *   *설명*: 클릭하면 숨겨져 있던 선택지(List)가 아래로 주르륵 펼쳐지는 메뉴.

## F
*   **FAB (Floating Action Button / 플로팅 액션 버튼)**
    *   *설명*: 화면 우측 하단 등에 고정되어 스크롤해도 계속 떠다니는(Floating) 동그란 포인트 버튼. 해당 페이지의 가장 핵심 액션(글쓰기 ✏️, 플러스 ➕)에 배정됨.

## H
*   **Hero Image / Hero Banner (히어로 뷰)**
    *   *설명*: 웹사이트 메인 페이지 최상단에 브라우저 가로를 꽉 채우며 나타나는 압도적이고 거대한 이미지/영상과 거대한 카피 문구 영역.

## I
*   **Input / TextField (인풋 / 텍스트 필드)**
    *   *설명*: 사용자가 직접 텍스트를 타이핑해서 넣을 수 있는 네모난 입력 창. (예: 아이디/비밀번호 입력칸).

## P
*   **Pagination (페이지네이션)**
    *   *설명*: 게시판 목록 하단에 `[<] 1 2 3 4 5 [>]` 형태로 페이지 번호를 나누어 보여주는 컨트롤.
*   **Popover (팝오버)**
    *   *설명*: 특정 요소를 클릭했을 때 그 요소 바로 근처에 꼬리를 달고 나타나는 부가적인 제어 패널(툴팁보다 복잡한 상호작용 가능).
*   **Progress Indicator (프로그레스 바 / 스피너)**
    *   *설명*: 데이터 로딩이나 작업 진행 상태를 보여주는 요소.
        *   `Linear`: 가로로 채워지는 막대 바.
        *   `Circular (Spinner)`: 뱅글뱅글 도는 원형 아이콘.

## R
*   **Radio Button (라디오 버튼)**
    *   *설명*: 여러 옵션 중 **'단 하나만 선택'**할 수 있을 때 사용하는 동그란 버튼. (하나를 누르면 다른 하나가 해제되는 옛날 라디오 버튼에서 유래).

## S
*   **Skeleton Screen (스켈레톤 UI)**
    *   *설명*: 데이터를 불러오기 전에, 실제 콘텐츠가 들어갈 자리(사진, 글 라인)에 회색 네모 박스를 깜빡거리게 두어 체감 로딩 속도를 줄이는 UI.
*   **Snackbar (스낵바) / Toast (토스트)**
    *   *설명*: 작업 완료/실패(예: "장바구니에 담겼습니다")를 알리고 **몇 초 뒤 자동으로 스르륵 사라지는** 알림창. 모바일은 주로 하단(스낵바), PC는 우측 상단(토스트)에 나타남.
*   **Stepper (스테퍼)**
    *   *설명*: `1.장바구니 > 2.결제정보 > 3.주문완료` 처럼 단계별 진행 상황을 점과 선으로 이어 보여주는 내비게이션.
*   **Switch / Toggle (스위치 / 토글)**
    *   *설명*: 알람 ON/OFF, 다크모드 설정 등 두 가지 상태를 딸깍딸깍 즉각적으로 껐다 켤 수 있는 모바일형 스위치.

## T
*   **Tabs (탭)**
    *   *설명*: 상단에 버튼들을 나란히 배치하여, 누를 때마다 동일한 공간의 화면 콘텐츠만 휙휙 전환해 보여주는 메뉴.
*   **Tooltip (툴팁)**
    *   *설명*: 아이콘 위에 마우스를 1초 정도 올려두면(Hover) 짧은 텍스트 설명표가 뿅 하고 나타나는 말풍선.
