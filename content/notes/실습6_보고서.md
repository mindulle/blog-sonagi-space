## 1. 실습 목적
P2P 네트워크 기반의 분산형 DB인 IPFS와 OrbitDB의 동작 원리를 이해하고, 노드 간 양방향 데이터 쓰기 권한 설정과 데이터 무결성을 위한 유효성 검사 로직을 구현한다.

## 2. 실습 환경 및 구현 내용
* **OS / Runtime**: Linux (Ubuntu), Node.js v25.6.1
* **양방향 동기화**: OrbitDB의 `accessController`에 `write: ['*']`를 적용하여 모든 노드가 데이터를 기록할 수 있도록 하였다.
* **유효성 검사**: `!/\d/.test(str)` 정규식을 활용하여 숫자가 포함된 데이터의 저장을 원천 차단하였다.

## 3. 실습 결과 및 테스트 데이터 (스크린샷)

### 3.1 Node 1 초기화 및 Node 2 양방향 동기화
* **Node 1 실행 (좌)**: IPFS 노드 생성 후 `PeerID`와 `DB address`가 발급되었으며, 초기 데이터 `name = Alice`가 저장되었다.
* **Node 2 실행 및 검증 (우)**: Node 1 접속 후 'Alice'를 동기화하였다. 이후 Node 2에서 'Bob'을 입력하자 양방향 동기화가 성공적으로 이루어졌으며, 숫자가 포함된 'Alice1'은 유효성 검증 로직에 의해 차단(`put rejected`)되었다.

## 4. 결론 및 고찰
IPFS와 OrbitDB를 활용한 P2P 데이터베이스 시스템의 핵심 메커니즘을 이해할 수 있었다. 중앙 서버가 없는 분산 환경에서는 적절한 Access Control(접근 제어)과 Data Validation(유효성 검사)이 시스템의 무결성을 유지하는 데 결정적인 역할을 한다는 점을 깨달았다.
