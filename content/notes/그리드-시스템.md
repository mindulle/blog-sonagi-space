# 그리드 시스템 (Grid System)

그리드 시스템은 디자인에서 시각적 요소들을 조화롭고 일관성 있게 배치하기 위해 사용하는 뼈대이자 가이드라인입니다. 정보를 구조화하고, 사용자의 시선을 효과적으로 유도하며, 다양한 디바이스 환경(반응형 웹)에서 안정적인 레이아웃을 유지하는 데 핵심적인 역할을 합니다.

## 1. 그리드의 역사: 스위스 디자인 (Swiss Style)
그리드 시스템이 현대적인 형태로 정립된 것은 1950년대 스위스를 중심으로 발전한 **국제 타이포그래피 양식(International Typographic Style)**, 일명 스위스 디자인 덕분입니다. 
장식을 배제하고 오직 수학적 비례, 산세리프 서체(예: Helvetica), 그리고 철저한 그리드를 활용하여 '가독성'과 '객관성'을 극대화한 것이 특징입니다. 이 철학이 현대 UI/UX 디자인의 근간이 되었습니다.

## 2. 그리드의 핵심 종류 및 시각 자료

### 2.1 매뉴스크립트 그리드 (Manuscript Grid)
<svg width="200" height="120" xmlns="http://www.w3.org/2000/svg"><rect width="200" height="120" fill="#f0f0f0" rx="4"/><rect x="25" y="20" width="150" height="80" fill="#8cb4ff" rx="2"/></svg>
- **개념**: 단행본 책이나 긴 텍스트 블록에 사용되는 가장 단순한 형태의 그리드. 블록 그리드(Block Grid)라고도 합니다.
- **특징**: 넓은 직사각형 판면 하나와 이를 둘러싼 여백(Margin)으로 구성됩니다.
- **용도**: 블로그 포스트, 소설, PDF 문서.

### 2.2 컬럼 그리드 (Column Grid)
<svg width="200" height="120" xmlns="http://www.w3.org/2000/svg"><rect width="200" height="120" fill="#f0f0f0" rx="4"/><rect x="20" y="20" width="35" height="80" fill="#8cb4ff" rx="2"/><rect x="65" y="20" width="35" height="80" fill="#8cb4ff" rx="2"/><rect x="110" y="20" width="35" height="80" fill="#8cb4ff" rx="2"/><rect x="155" y="20" width="35" height="80" fill="#8cb4ff" rx="2"/></svg>
- **개념**: 화면을 세로 형태의 여러 단(Column)으로 나눈 그리드입니다.
- **특징**: 웹사이트나 잡지에서 가장 흔하게 쓰입니다. 모바일은 4단, 태블릿은 8단, 데스크탑은 12단 그리드를 사용하는 것이 UI 디자인의 표준처럼 여겨집니다.
- **용도**: 뉴스 사이트, 웹사이트 본문, 매거진.

### 2.3 모듈러 그리드 (Modular Grid)

- **개념**: 컬럼 그리드를 가로선(Flowline)으로 한 번 더 쪼개어, 동일한 크기의 '모듈(Module)'이라는 사각형 칸들을 만든 그리드입니다.
- **특징**: 가로/세로 정렬이 모두 가능하여 훨씬 복잡하고 다양한 레이아웃을 통일성 있게 구성할 수 있습니다.
- **용도**: 이커머스 상품 목록, 핀터레스트, 갤러리 앱, 대시보드.

### 2.4 베이스라인 그리드 (Baseline Grid)
<svg width="200" height="120" xmlns="http://www.w3.org/2000/svg"><rect width="200" height="120" fill="#f0f0f0" rx="4"/><line x1="10" y1="30" x2="190" y2="30" stroke="#8cb4ff" stroke-width="2" stroke-dasharray="4 2"/><line x1="10" y1="50" x2="190" y2="50" stroke="#8cb4ff" stroke-width="2" stroke-dasharray="4 2"/><line x1="10" y1="70" x2="190" y2="70" stroke="#8cb4ff" stroke-width="2" stroke-dasharray="4 2"/><line x1="10" y1="90" x2="190" y2="90" stroke="#8cb4ff" stroke-width="2" stroke-dasharray="4 2"/></svg>
- **개념**: 텍스트의 밑선(Baseline)을 맞추기 위한 수평선 위주의 그리드입니다. 노트의 줄처럼 작동합니다.
- **특징**: 주로 4pt나 8pt 단위의 배수를 사용하여, 모든 텍스트와 UI 요소의 상하 간격이 일정한 리듬감을 가지도록 돕습니다.
- **용도**: 타이포그래피 중심의 에디토리얼 디자인, 밀도 높은 UI 시스템.

## 3. 웹 퍼블리싱 관점에서의 그리드 (코드 구현)

현대의 디지털 프로덕트에서 그리드 시스템은 코드로 직접 구현되며, 디자이너와 개발자 간의 약속입니다.

### 3.1 CSS Grid (12-Column 레이아웃 예시)
Figma에서 설정한 데스크탑 12단 그리드를 CSS로 완벽하게 구현하는 방법입니다.

```css
/* 부모 컨테이너에 12단 컬럼 그리드 설정 */
.grid-container {
  display: grid;
  
  /* 12개의 컬럼을 생성하고 남은 공간(1fr)을 균등 분배 */
  grid-template-columns: repeat(12, 1fr);
  
  /* 거터(Gutter): 컬럼과 컬럼 사이의 간격을 24px로 설정 (8pt Grid 기반) */
  gap: 24px;
  
  /* 마진(Margin): 화면 양옆 여백 */
  padding: 0 5%;
}

/* 자식 요소(콘텐츠) 배치 예시 */
.sidebar {
  /* 1번째 라인부터 4칸 차지 (보통 사이드바) */
  grid-column: span 4;
}

.main-content {
  /* 남은 8칸 차지 (메인 콘텐츠 영역) */
  grid-column: span 8;
}
```

### 3.2 8pt Grid System (CSS 변수 활용 예시)
UI 시스템에서 간격(Spacing)을 통일하기 위해 CSS 변수로 8의 배수를 설정하여 사용하는 실무 방식입니다.

```css
:root {
  /* 8pt Grid System 베이스라인 설정 */
  --space-1: 8px;
  --space-2: 16px;
  --space-3: 24px;
  --space-4: 32px;
  --space-5: 40px;
  --space-6: 48px;
}

.card-module {
  padding: var(--space-3); /* 내부 패딩 24px */
  margin-bottom: var(--space-4); /* 아래 간격 32px */
  border-radius: var(--space-1); /* 둥근 모서리 8px */
}
```

## 4. UI/UX 디자인 시스템의 3요소
디지털 환경에서 반응형 웹 그리드를 세팅할 때 반드시 정의해야 하는 3가지입니다.
1. **Columns (컬럼)**: 콘텐츠가 배치되는 세로 기둥 (예: 데스크탑 12개, 모바일 4개).
2. **Gutters (거터)**: 컬럼과 컬럼 사이의 여백 (예: 데스크탑 24px, 모바일 16px).
3. **Margins (마진)**: 전체 화면 양 끝의 바깥 여백.
