# 📖 프로그래밍언어론 기말고사 대비 핵심 요약 노트
**사용법**: 기출문제(Master Kit)나 웹 모의고사를 풀다가 틀렸을 때, 해당 단원으로 돌아와 개념을 다시 정립하는 **개념 백과사전**입니다.

---

## 📌 단원 1. 바인딩과 스코프 (Binding & Scope)

### 1.1 이름과 별칭 (Names & Aliases)
* **L-value (Location)**: 변수의 **메모리 주소**.
* **R-value (Read/Value)**: 변수에 저장된 실제 **데이터 값**.
* **별칭 (Alias)**: 같은 메모리 주소(L-value)를 둘 이상의 이름이 참조하는 현상.
  * *발생 원인*: C/C++의 공용체(`union`), Fortran의 `EQUIVALENCE`, 포인터(`*`), 참조 전달(Call-by-Reference).
  * *단점*: 의도치 않은 값의 변경(Side Effect)으로 인해 프로그램 가독성과 신뢰성이 크게 떨어짐.

### 1.2 바인딩 시각 (Binding Time)
이름이나 속성이 결정되는 '시점'을 뜻하며, 빠를수록 효율적이고 느릴수록 유연합니다.
1. **언어 정의 시점**: 연산자(`+`, `-`)의 의미 결정.
2. **언어 구현 시점**: `int` 타입의 크기(4바이트 등) 결정.
3. **컴파일 시점 (Translation)**: 정적 변수의 타입 결정.
4. **링크/로드 시점 (Link/Load)**: 외부 모듈 연결 및 메모리 주소 할당.
5. **실행 시점 (Execution/Runtime)**: 지역 변수에 실제 값이 바인딩되는 시점.

### 1.3 스코프 (Scope)와 참조 환경
* **정적 스코프 (Static / Lexical Scope)**:
  * 변수의 영역이 **소스 코드의 물리적(공간적) 구조**에 의해 컴파일 타임에 결정됨. (C, Java 등 대부분의 현대 언어)
  * **영역 구멍 (Scope Hole)**: 안쪽 블록에 똑같은 이름의 지역변수가 선언되어, 바깥쪽(전역) 변수가 가려지는 현상. (C++의 `::x` 범위 지정 연산자로 접근 가능)
* **동적 스코프 (Dynamic Scope)**:
  * 변수의 영역이 서브프로그램의 **호출 순서(시간적 흐름)**에 의해 실행 시간에 결정됨.
  * `sub1()`에서 변수를 못 찾으면 자신을 부른 **호출자(Caller)**의 환경을 찾아 올라감. 정적 타입 검사가 불가능하여 신뢰성이 떨어짐.
* **참조 환경 (Referencing Environment)**: 프로그램의 특정 지점에서 실제로 참조(사용)할 수 있는 모든 이름의 집합.

## 📌 단원 2. 공리적 의미론 (Axiomatic Semantics & Hoare Logic)

### 2.1 공리적 의미론 기초
* 프로그램의 상태 변화를 **사전조건(Precondition)**과 **사후조건(Postcondition)**의 논리적 관계로 증명하는 방식.
* **최약 사전조건 (Weakest Precondition, wp)**: 주어진 사후조건(Q)을 만족하기 위해 프로그램 실행 전 보장되어야 하는 가장 덜 제한적인 조건.

### 2.2 Hoare Logic의 주요 규칙
1. **대입 공리 (Assignment Axiom)**: `wp(x := E, Q) = Q[E/x]`
   * 사후조건 Q의 변수 x 자리에 식 E를 그대로 대입하여 계산함.
2. **귀결 규칙 (Rule of Consequence)**:
   * **사전조건은 강화(Strengthen)**하고, **사후조건은 약화(Weaken)**해도 증명은 여전히 성립함. (더 빡빡한 조건에서 시작해서 덜 엄격한 결과를 기대하는 것은 항상 참이기 때문)
3. **루프 불변식 (Loop Invariant)**:
   * 반복문(`while`) 증명의 핵심. **①반복 전, ②반복 중, ③반복 후** 세 시점에서 모두 항상 **참(True)**이어야 하는 논리식.

### 2.3 다익스트라의 조건부 명령어 (Guarded Commands)
* `if <cond> -> <stmt> [] <cond> -> <stmt> fi`
* **비결정성 (Nondeterminism)**: 여러 조건이 참일 때, 위에서부터 순서대로 실행하는 것이 아니라 **무작위로 하나를 선택**하여 실행함. 프로그램 논리 증명을 수월하게 하기 위해 고안됨.

## 📌 단원 3. 논리형 프로그래밍 (Prolog)

### 3.1 프롤로그의 기반
* 1972년 Alain Colmerauer 개발.
* **Robert Kowalski**: `Algorithm = Logic + Control` 제창 (선언적 프로그래밍의 정수).
* 1차 술어 논리를 기반으로 하며, 모든 문장은 **혼 절 (Horn Clause)**의 형태를 띰. (사실, 규칙, 질의)

### 3.2 문법과 동작 원리
* **문법 주의사항**:
  * **상수(Constant) & 술어**: 반드시 **소문자**로 시작. (예: `prolog`, `apple`)
  * **변수(Variable)**: 반드시 **대문자**나 **밑줄(`_`)**로 시작. (예: `X`, `_Name`)
  * 모든 선언은 반드시 **마침표(온점, `.`)**로 끝나야 함.
* **해답 탐색 메커니즘**:
  1. **반박 (Refutation)**: 증명하고자 하는 질의(Goal)를 일단 "거짓이다"라고 가정한 뒤, 시스템 내의 지식들을 바탕으로 모순(Contradiction)을 찾아내어 원래 질의가 "참"임을 증명하는 방식.
  2. **동형화 (Unification)**: 양쪽 논리식(항)이 똑같아지도록 변수와 상수를 매칭하여 묶어주는(Binding) 패턴 매칭 과정. (예: `food(X)`와 `food(apple)`에서 `X=apple`로 바인딩).

## 📌 단원 4. 서브프로그램과 매개변수 (Subprograms & Parameters)

### 4.1 매개변수 전달 방식 (Parameter Passing) - 🌟 100% 출제
| 전달 방식 (Call-by) | 의미 모델 (방향) | 설명 및 특징 | 부작용 (Alias 등) |
| :--- | :--- | :--- | :--- |
| **Value (값)** | In (입력) | 실인수의 '값'만 형식인수로 복사. | 배열 등 큰 데이터 복사 시 오버헤드. 원본은 안전함. |
| **Result (결과)** | Out (출력) | 함수 종료 시 지역변수의 결과값을 실인수로 복사. | 매개변수 평가 시점에 따라 값 충돌 발생 가능. |
| **Value-Result (값-결과)** | In-Out (입출력) | 시작 시 복사(in), 종료 시 복사(out). | **별칭(Alias)이 발생하지 않음**이 최대 장점. |
| **Reference (참조)** | In-Out (입출력) | 실인수의 '주소(경로)'를 그대로 전달. 효율적임. | **별칭(Alias)이 무조건 발생**하여 부작용 위험 높음. |
| **Name (이름)** | 식 자체 전달 | 인수 자체가 텍스트로 치환됨 (**Late Binding / 늦은 바인딩**). | 사용될 때마다 매번 새로 평가됨. **젠슨의 장치** 구현 가능. |

### 4.2 활성화 레코드 (Activation Record / Stack Frame)
서브프로그램이 호출될 때 스택(Stack)에 동적으로 할당되는 데이터 구조.
* **구성 요소**: 매개변수, 지역 변수, 복귀 주소(Return Address), 정적/동적 링크.
* **동적 링크 (Dynamic Link)**: 자신을 **실제로 호출한 함수(Caller)**의 활성화 레코드를 가리킴. 함수가 끝난 뒤 어디로 돌아갈지(Call Stack 추적)에 사용됨.
* **정적 링크 (Static Link)**: 텍스트 구조 상 자신을 감싸고 있는 **정적 부모(Static Parent)**의 활성화 레코드를 가리킴. 정적 스코프에서 비지역 변수(non-local)를 탐색할 때 사용됨.

### 4.3 특수 서브프로그램
* **코루틴 (Coroutine)**: 일반적인 서브프로그램(Master-Slave 종속적 관계)과 달리, 진입 위치(Entry point)가 여러 개이며 대칭적(Symmetric) 구조를 가짐. 서로가 서로의 실행을 중단하고 **재개(Resume)**하며 유사 병렬성(Quasi-concurrent)을 가짐.
* **제네릭 (Generic / Parametric Polymorphism)**: 타입을 매개변수화 하여 다양한 타입에 재사용할 수 있는 함수. (C++의 `template`). 컴파일 시점에 구체적인 타입으로 인스턴스화됨.
