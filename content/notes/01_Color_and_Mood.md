# 🎨 Vol 1. 색상과 무드 (Color & Mood)

색상은 디자인 피드백에서 가장 추상적이고 주관적인 감정 표현이 쏟아지는 영역입니다. 이 볼륨은 현업 디자이너들이 흔히 쓰는 '느낌적인 느낌'의 컬러 은어들을 색채학(Color Theory)과 명도/채도(Value/Chroma) 기반의 시스템 용어로 완벽하게 치환합니다.

---

## 1. 선명도 및 대비 (Clarity & Contrast)

### 1.1 "쨍하다" / "팝(Pop)하다" / "튀어나온다"
*   **시스템 용어**: High Chroma (고채도), High Contrast (고대비), Vibrant
*   **현상**: 특정한 색상의 채도가 매우 높고 주변 배경색과의 명도 대비가 강해 시선이 강력하게 집중되는 상태.
*   **실무 솔루션**:
    *   HSB 컬러 모델에서 **S(Saturation, 채도)** 값을 80~100%로 끌어올림.
    *   포인트로 쓸 부분의 면적을 전체 화면의 5~10% 미만으로 줄여야 진정한 '쨍함'이 살아남 (면적이 넓어지면 '눈 아프다'로 바뀜).

### 1.2 "탁하다" / "칙칙하다" / "구리다"
*   **시스템 용어**: Muddy, Low Chroma (저채도), Desaturated, Unbalanced Value
*   **현상**: 색상에 무채색(회색/검은색)이 애매하게 섞여 있거나, 배경과 전경의 명도(Lightness) 차이가 없어서 경계가 흐리멍텅할 때 발생. 특히 원색에 투명도(Opacity)를 주고 어두운 배경에 올렸을 때 흔히 발생함.
*   **실무 솔루션**:
    *   색상의 명도(Value)를 조절해 대비를 확실히 주거나, 애매한 중간 톤을 버리고 완전히 밝거나 완전히 어두운 톤으로 이동.
    *   블렌딩 모드(Blending Mode)를 점검하여 회색조가 섞이지 않도록 함.

### 1.3 "묻힌다" / "먹힌다" / "안 보인다"
*   **시스템 용어**: Low Contrast Ratio (명도 대비 실패), Accessibility Failure
*   **현상**: 버튼 위의 글자색과 버튼 배경색의 명도(밝기)가 비슷해서 글자가 배경에 흡수된 것처럼 보이는 현상.
*   **실무 솔루션**:
    *   WCAG 2.1 접근성 기준에 따라 텍스트와 배경의 명도 대비를 최소 **4.5:1** (큰 글자는 3:1) 이상으로 강제 조정.

### 1.4 "눈 아프다" / "눈이 시리다" / "너무 튄다"
*   **시스템 용어**: Screaming, Clashing Colors, Vibrating Boundaries (경계 진동)
*   **현상**:
    *   1) 채도가 100%인 야광색(Neon)을 넓은 면적에 바른 경우.
    *   2) 명도가 비슷한 보색(예: 새파란 배경에 새빨간 글씨)을 겹쳐 두어 경계선이 일렁이듯 착시(Vibrating boundaries)를 일으키는 경우.
*   **실무 솔루션**: 두 색상 사이에 얇은 무채색(흰색/검은색) 선을 넣거나, 한쪽 색상의 채도와 명도를 대폭 낮춰(Desaturate) 배경으로 강등시킴.

## 2. 조화와 무드 (Harmony & Atmosphere)

### 2.1 "붕 뜬다" / "따로 논다"
*   **시스템 용어**: Unanchored, Color Palette Mismatch
*   **현상**: 전체 UI는 웜톤(따뜻한 색)인데 특정 버튼이나 일러스트만 쿨톤(차가운 색)이거나, 시스템 팔레트에 없는 엉뚱한 색(Out-of-gamut)이 난입하여 시각적으로 연결되지 않는 상태.
*   **실무 솔루션**: 디자인 토큰(Design Tokens)에 정의된 색상 변수(예: `var(--primary-500)`)로 색상을 덮어씌움. 다른 사진이나 에셋이라면 이미지 위에 캘리브레이션용 컬러 오버레이(Color Overlay)를 살짝 씌워 톤을 맞춤.

### 2.2 "톤온톤(Tone on Tone)으로 빼주세요"
*   **의미**: 색상(Hue)은 동일하게 유지하되, **명도(밝기)와 채도만 다르게** 조합하는 기법.
*   **효과**: 매우 차분하고 세련되며 안정적인 무드 연출. (예: 진파랑 배경에 연한 파랑 글씨).

### 2.3 "톤인톤(Tone in Tone)으로 갑시다"
*   **의미**: 색상(Hue)은 서로 다르게 쓰되, **명도와 채도(톤)를 동일하게** 맞추는 기법.
*   **효과**: 파스텔톤, 빈티지톤처럼 전체적인 분위기(Vibe)를 통일하면서도 다채로운 느낌을 줌.

### 2.4 "듀오톤(Duotone)으로 포인트 줄까요?"
*   **의미**: 단 두 가지의 색상(또는 같은 색상의 투명도 차이)만 사용하여 아이콘이나 이미지를 표현하는 기법.
*   **실무 솔루션**: 플랫 아이콘을 그릴 때, 메인 뼈대(Stroke)는 100% 진한 색으로 잡고, 내부의 빈 공간(Fill)이나 장식 요소는 **동일한 색상의 10~20% 투명도(Opacity)**로 채워 넣어 입체감과 세련미를 줌. (최근 UI 트렌드에서 가장 많이 쓰이는 아이콘 스타일).

### 2.5 "색이 죽었어요" / "탁기가 도네요"
*   **의미**: 화면을 캡처하거나 다른 툴로 옮겼을 때, 컬러 프로파일(RGB vs CMYK, sRGB vs Display P3) 차이로 인해 쨍했던 색이 칙칙한 회색빛을 띠게 된 치명적인 에러 상태.

---

## 3. 심도와 무게 (Depth & Weight)

### 3.1 "묵직하게 눌러주세요"
*   **시스템 용어**: Darken, Add Weight, Decrease Lightness
*   **현상/요구**: 화면이 너무 밝은 색 위주라 가벼워 보일 때, 신뢰감이나 안정감을 주기 위해 베이스 컬러의 명도를 확 낮추는 작업.
*   **실무 솔루션**: 검은색을 섞는 대신(탁해짐), 색상(Hue)을 약간 차가운 계열(Blue/Purple)로 틀면서 명도를 낮추는 **'리치 블랙(Rich Black / Slate)'** 기법 사용. (예: `#000000` 대신 `#0f172a` 사용).

### 3.2 "가볍고 시원하게 뚫어주세요"
*   **시스템 용어**: Lighten, Airy, High Key
*   **현상/요구**: 여백(White space)을 대폭 늘리고 배경색을 화이트나 아주 옅은 쿨톤 그레이(`#f8fafc`)로 빼서 쾌적한 공간감을 조성하라는 뜻.

## 4. [Data Sourced] Satori Graphics 특화 컬러 은어
*   **"Push the hex code down / Needs Contrast"**
    *   *상황*: 라이트 그레이 배경에 그레이 텍스트처럼 명도 차이가 없을 때.
    *   *액션*: 헥스코드(Hex)의 명도(Lightness) 값을 뚝 떨어뜨려(Push down) 대비를 강제로 벌림.
*   **"Harmonize the Saturation"**
    *   *상황*: 두 색상이 어울리지 않고 충돌(Clashing)할 때.
    *   *액션*: 색상(Hue)은 달라도 채도(Saturation) 레벨을 비슷한 %로 맞춰주면 시각적 조화(Harmony)가 생김.
