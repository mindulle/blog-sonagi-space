# 🔗 As-Is Specs: Community & Extensions (Hot Deal, Beggar Room)

이 문서는 실제 서비스인 `거지맵.com`에서 관찰된 외부 정보 연동 및 커뮤니티 연결 로직을 역기획하여 작성되었습니다.

---

### 1. 핫딜(Hot Deal) 시스템

#### 1.1 정보 큐레이션 및 노출
*   **성격**: 앱 내부의 POI 정보가 아닌, 외부 이커머스(쿠팡, 11번가, 알리 등)의 파격적인 할인 상품 정보를 제공한다.
*   **정보 구성**:
    *   상품 이미지 및 명칭
    *   정가 대비 할인율 및 최종 할인가
    *   마감 임박 여부 (Time-sensitive label)
*   **업데이트 방식**: 운영자('왕초')가 직접 선별하여 등록하거나, 제휴 API를 통해 자동 갱신되는 구조로 추정된다.

#### 1.2 외부 랜딩 로직 (Affiliate Routing)
*   **Deep Link 연동**: 사용자가 핫딜 아이템을 클릭하면 해당 쇼핑몰 앱 혹은 웹사이트의 상품 페이지로 즉시 이동한다.
*   **수수료 구조**: 대부분의 링크가 제휴 마케팅(Affiliate) 코드를 포함하고 있어, 서비스의 주요 수익 모델 중 하나로 기능한다.

---

### 2. 거지방(Beggar Room) 연결 시스템

#### 2.1 커뮤니티 가교 역할
*   **정의**: 앱 내에서 직접적인 채팅 기능을 제공하는 대신, 이미 활성화된 외부 커뮤니티(카카오톡 오픈채팅방 등)로 사용자를 연결한다.
*   **카테고리화**: '서울 거지방', '대학생 짠테크방' 등 지역이나 관심사별로 분화된 외부 링크 리스트를 제공한다.

#### 2.2 연결 흐름 (User Flow)
*   **진입**: 네비게이션 바 혹은 메인 홈의 '거지방' 버튼 클릭.
*   **이동**: 클릭 시 "카카오톡으로 이동하시겠습니까?" 등의 안내 후 외부 앱 실행.
*   **효과**: 서버 유지비가 많이 드는 채팅 기능을 직접 구현하지 않고도, 강력한 사용자 소속감과 실시간 정보 교환 환경을 구축한다.

### 3. 데이터 요구사항 (As-Is Fields)

| 필드명 | 설명 | 비고 |
| :--- | :--- | :--- |
| `deal_id` | 핫딜 고유 ID | |
| `affiliate_url` | 제휴 코드가 포함된 외부 쇼핑몰 링크 | 클릭 시 라우팅 대상 |
| `discount_rate` | 할인율 (%) | 시각적 강조용 |

### 💡 서비스 총평 (Reverse Engineering Conclusion)
거지맵의 서비스 구조는 **"복잡한 기능(지도 API, 포인트 시스템)은 내부에 집중하고, 소통과 수익화(채팅, 핫딜)는 외부 플랫폼을 영리하게 활용"**하는 전형적인 고효율 MVP 모델을 따르고 있습니다.
