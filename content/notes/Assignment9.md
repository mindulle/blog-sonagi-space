# Assignment 9: Prolog로 작성한 GCD 및 LCM

**학번:** 201524471  
**성명:** 박혜성

---

## 1. 문제
두 개의 양의 정수를 입력받아, 그 두 수의 최대공약수(GCD)와 최소공배수(LCM)를 계산한 후, 두 값의 차이(LCM - GCD)를 출력하는 Prolog 프로그램을 작성하는 것입니다. 이때 SWI-Prolog의 내장 `gcd` 함수를 사용하지 않고 직접 구현해야 하며, 구현한 술어(predicate)에는 자신의 이름 약자(HP)를 접두사로 붙여야 합니다.

## 2. 전략
문제 해결을 위해 다음과 같은 전략을 사용하였습니다.

1.  **최대공약수(GCD) 구현**: 유클리드 호제법의 뺄셈 방식을 사용하여 `hp_gcd/3` 술어를 구현하였습니다.
    -   `hp_gcd(0, Y, Y)`: 한 숫자가 0이 되면 다른 숫자가 최대공약수입니다.
    -   `X < Y`인 경우 순서를 바꾸어 재귀 호출합니다.
    -   `X >= Y`인 경우 `X - Y`를 계산하여 재귀 호출합니다.
2.  **최소공배수(LCM) 구현**: GCD를 활용하여 최소공배수를 구하는 공식을 사용하였습니다.
    -   `LCM(X, Y) = (X * Y) / GCD(X, Y)`
    -   이 계산을 수행하는 `hp_lcm/3` 술어를 구현하였습니다.
3.  **메인 루프 구성**: `initialization(main, main)`을 사용하여 프로그램 실행 시 자동으로 `main` 술어가 호출되도록 하였습니다. `main`에서는 `read/1`을 통해 입력을 받고, 계산 결과를 `writeln/1`로 출력한 후 `halt/0`으로 종료합니다.

## 3. 언어의 모듈 기능
Prolog에서는 대규모 프로젝트의 코드 관리를 위해 모듈 시스템을 제공합니다.

-   **모듈 정의**: 파일 상단에 `:- module(모듈이름, [내보낼술어목록]).` 형식을 사용하여 정의합니다.
-   **활용 방법**: 만약 GCD와 LCM 기능을 별도의 모듈로 분리했다면, 각각 `hp_gcd.pl`과 `hp_lcm.pl`로 저장하고 `module` 선언을 통해 필요한 술어만 외부로 공개(export)할 수 있습니다.
-   **이점**: 이름 충돌을 방지하고 코드의 재사용성을 높일 수 있습니다. 예를 들어, `hp_lcm` 모듈에서 `hp_gcd` 모듈을 불러올 때는 `:- use_module(hp_gcd).`와 같이 선언하여 사용할 수 있습니다.

## 4. 토의
Prolog에서 모듈을 사용하는 다른 방법으로는 다음과 같은 것들이 있습니다.

1.  **consult/1 사용**: `consult('file.pl').` 또는 `['file.pl'].`을 사용하여 다른 파일의 내용을 현재 환경으로 불러올 수 있습니다. 이는 모듈 시스템보다는 단순한 파일 포함(inclusion)에 가깝습니다.
2.  **명시적 모듈 참조**: `module:predicate(Args)` 형식을 사용하여 특정 모듈에 속한 술어를 명시적으로 호출할 수 있습니다.
3.  **라이브러리 활용**: SWI-Prolog에서 제공하는 수많은 라이브러리(`:- use_module(library(lists)).` 등)들도 이 모듈 시스템을 기반으로 동작합니다.

## 5. 결론
이번 과제를 통해 Prolog에서 재귀를 활용한 알고리즘 구현 방법과 입출력 처리 방식을 익힐 수 있었습니다. 특히 `initialization/2`를 사용하여 독립 실행 가능한 프로그램을 만드는 법과, 모듈 시스템을 통한 코드 구조화의 중요성을 배울 수 있었던 유익한 실습이었습니다. 유클리드 호제법을 직접 구현해보며 Prolog의 패턴 매칭과 백트래킹 특성을 더 깊이 이해하게 되었습니다.
