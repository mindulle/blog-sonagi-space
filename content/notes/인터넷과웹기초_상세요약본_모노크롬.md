## 1. 반응형 웹과 CSS 레이아웃
모바일, 태블릿, 데스크탑 등 다양한 해상도에 맞게 유동적으로 변하는 웹을 설계하기 위한 핵심 기술이다.

### 1.1 Viewport 설정
반응형 웹의 뼈대. 브라우저가 화면의 렌더링 영역을 기기의 실제 물리적 너비에 맞추도록 강제한다.
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

### 1.2 미디어 쿼리 (Media Query)
해상도나 기기 방향에 따라 다른 CSS를 적용한다.
* **문법**: `@media screen and (min-width: 768px) { ... }`
* **모바일 퍼스트**: 모바일 CSS를 기본으로 작성하고, `min-width`를 통해 데스크탑 레이아웃으로 확장해 나가는 방식. (현대 웹 표준 권장)
* **데스크탑 퍼스트**: PC CSS를 기본으로 하고, `max-width`로 모바일에 맞게 좁혀가는 방식.

### 1.3 가변 길이 단위 (Units)
* `vw` / `vh`: 뷰포트(브라우저 창)의 가로/세로 길이의 1%를 의미한다.
* `em`: 자신의 부모 요소 폰트 크기를 1배수로 기준 삼는다. 자식 요소로 깊어질수록 폰트 크기가 곱해져 비정상적으로 커지거나 작아지는 **중첩(Compounding) 문제**가 발생할 수 있다.
* `rem` (root em): 오직 문서의 최상위 태그인 `<html>`의 폰트 크기만을 기준으로 한다. 레이아웃의 수학적 일관성을 유지할 수 있어 가장 널리 쓰인다.

## 2. 모던 자바스크립트 문법 (ES6+)
자바스크립트의 유연함으로 인해 발생하는 오류를 막기 위해 추가된 엄격한 문법들.

### 2.1 변수 선언 (var, let, const)
* `var`: 함수 스코프(Function Scope). 같은 이름의 변수를 재선언해도 에러가 나지 않아 예기치 않은 버그를 유발한다.
* `let`: 블록 스코프 `{}`. 변수의 재할당은 가능하나 재선언은 불가하다.
* `const`: 블록 스코프 `{}`. 상수이므로 재할당이 절대 불가하다. 단, 객체나 배열을 `const`로 선언했을 때 **내부 속성(Property)이나 요소는 추가/수정이 가능**하다는 점에 유의한다.

### 2.2 호이스팅(Hoisting)과 TDZ
자바스크립트 엔진은 코드를 실행하기 전 모든 선언문(변수, 함수)을 메모리 최상단으로 끌어올리는 호이스팅을 수행한다.
* `var`는 끌어올려지면서 `undefined`로 초기화되어 선언 전에 호출해도 에러가 나지 않는다.
* `let`과 `const` 역시 호이스팅되지만, 선언 코드를 만나기 전까지는 초기화되지 않는 **일시적 사각지대(TDZ, Temporal Dead Zone)**에 빠지게 되어 참조 시 `ReferenceError`가 발생한다.

### 2.3 동등/일치 연산자와 Truthy/Falsy
* `==` (동등): 두 값의 타입이 다르면 강제로 형 변환을 하여 값만 비교한다. (`0 == ""` $\rightarrow$ `true`)
* `===` (일치): 값과 메모리 타입까지 엄격하게 비교한다. (실무 필수)
* **Falsy 값**: 조건문에서 `false`로 취급되는 값은 오직 6개뿐이다. (`0`, `""`, `null`, `undefined`, `NaN`, `false`)
* **함정**: `"false"`(문자열), `[]`(빈 배열), `{}`(빈 객체)는 모두 내용물의 유무를 떠나 객체 자체가 존재하므로 Truthy로 평가된다.

## 3. 배열 메서드와 화살표 함수

### 3.1 배열 제어
* **원본 배열을 직접 변경 (Mutating)**
  * `push()`: 맨 뒤에 추가 / `pop()`: 맨 뒤 삭제
  * `unshift()`: 맨 앞에 추가 / `shift()`: 맨 앞 삭제
* **새로운 배열을 반환 (Non-mutating)**
  * `map()`: 배열의 모든 요소를 순회하며 콜백 함수의 반환값으로 매핑된 새 배열 생성.
  * `filter()`: 콜백 조건식이 `true`인 요소들만 필터링하여 새 배열 생성.
  * **체이닝(Chaining)**: `.filter(x => x > 5).map(x => x * 2)` 처럼 메서드를 연속해서 호출하는 기법.

### 3.2 화살표 함수 (Arrow Function)
`function` 키워드 대신 `=>` 기호를 사용하여 함수를 간결하게 표현한다.
* 인자가 1개일 때는 소괄호 `()` 생략 가능.
* 본문이 한 줄의 `return` 문일 때는 중괄호 `{}`와 `return` 키워드 생략 가능.
* **주의**: 화살표 함수에서 객체를 암묵적으로 반환하려면 소괄호로 래핑해야 한다. `const getObj = () => ({ id: 1 })`

## 4. DOM 제어와 이벤트 모델

### 4.1 요소 탐색과 조작
* **탐색**: `getElementById()`, `querySelector()`. 인접 형제 요소를 찾을 때는 `nextElementSibling`을 사용.
* **텍스트 삽입의 차이**:
  * `innerHTML`: 전달된 문자열 내의 HTML 태그를 실제 DOM으로 해석하여 렌더링한다. (보안 취약점 XSS 원인)
  * `innerText` / `textContent`: 태그를 해석하지 않고 오직 순수 텍스트로만 화면에 노출시킨다.

### 4.2 이벤트 버블링과 차단
* **버블링(Bubbling)**: 자식 요소에서 발생한 이벤트가 부모 요소들로 차례대로 거슬러 올라가며 전파되는 현상.
* `e.stopPropagation()`: 버블링 현상을 인위적으로 끊어, 부모 요소로 이벤트가 전달되는 것을 막는다.
* `e.preventDefault()`: 폼의 제출(Submit)이나 `<a>` 태그의 페이지 이동 등 브라우저에 지정된 **기본 동작을 취소**시킨다. 전파를 막는 것과는 다른 개념이다.

### 4.3 이벤트 위임 (Event Delegation)
동적으로(JS로) 나중에 화면에 추가된 자식 요소에는 미리 `addEventListener`를 걸어둘 수 없다. 이를 해결하기 위해, **이미 존재하고 있는 상위 부모 요소**에 하나의 이벤트 리스너를 달아둔다.
* `e.target`: 마우스가 실제로 클릭한 가장 안쪽의 타겟 요소.
* `e.currentTarget`: 이벤트 리스너가 직접 부착되어 있는 부모 요소.

## 5. 실전 jQuery 라이브러리

### 5.1 계층 탐색
* `.find('.item')`: 선택된 요소의 하위(자손) 요소 중에서 해당 클래스를 모두 찾는다.
* `.closest('.card')`: 선택된 요소부터 시작하여, DOM 트리를 위쪽(조상)으로 거슬러 올라가며 **가장 먼저 일치하는 단 하나의 부모**를 찾는다. (이벤트 위임 패턴에서 핵심 역할)

### 5.2 속성 제어 (attr vs prop)
* `.attr()`: HTML 마크업에 적혀있는 초기 속성(Attribute) 값을 문자열 형태로 가져온다.
* `.prop()`: 체크박스의 `checked` 상태나 폼 요소의 활성화 여부 등 메모리 상에서 동적으로 변하는 **상태값(Boolean)**을 제어할 때 사용한다.

## 6. 비동기 통신과 JSON 데이터

### 6.1 JSON (JavaScript Object Notation)
자바스크립트의 객체 문법과 유사하나 매우 엄격하다.
* **문법 규칙**: 모든 키(Key)와 문자열(String) 값은 **반드시 큰따옴표("")**로 감싸야 한다. 작은따옴표나 후행 쉼표(Trailing Comma)는 파싱 에러를 유발한다.
* `JSON.stringify(obj)`: JS 객체를 전송용 JSON 문자열로 변환 (직렬화).
* `JSON.parse(str)`: 수신한 JSON 문자열을 해석해 JS 객체로 복원 (파싱).

### 6.2 Promise와 비동기 상태
* **상태**: `Pending`(대기중), `Fulfilled`(이행 성공 $\rightarrow$ `.then()`), `Rejected`(거부/실패 $\rightarrow$ `.catch()`)

### 6.3 Fetch API와 async/await
* `fetch()`는 비동기적으로 네트워크 요청을 수행하고 Promise를 반환한다. JSON 바디를 객체로 변환하기 위해 `.json()`을 호출해 기다려야 한다.
* `await` 키워드는 **반드시 `async` 로 선언된 함수 내부에서만** 사용할 수 있다.
* 네트워크 장애 대처를 위해 `try ... catch` 블록으로 감싸는 것이 실무 표준이다.
