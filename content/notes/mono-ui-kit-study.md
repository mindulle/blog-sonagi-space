# 📚 Study: MONO User Interface Kit

이 문서는 극도의 미니멀리즘과 텍스트 중심의 설계를 특징으로 하는 **MONO UI Kit**을 분석한 리포트입니다. 소나기(Sonagi) v3.0의 **'Thin'** 레이어(장식 배제, 가독성 중심)의 시각적 고결함을 완성하기 위한 레퍼런스로 작성되었습니다.

## 2. 💡 Key Insights for Sonagi v3.0

### 2.1 Pure Typography Layouts
- **분석**: 아이콘이나 그래픽 장식 없이도 폰트의 크기, 굵기, 행간만으로 완벽한 가독성을 구현합니다.
- **소나기 v3.0 적용**: 위키 본문 리딩 경험에서 'Thin' 레이어의 핵심은 **"읽기 방해 금지"**입니다. MONO의 타이포그래피 스케일을 벤치마킹하여, 기술 문서의 텍스트가 시각적으로 가장 아름답게 보일 수 있는 최적의 비율을 도출합니다.

### 2.2 Boundary through Spacing
- **분석**: 선(Border)을 사용하지 않고 오직 간격(Spacing)만으로 영역을 구분하는 고난도의 레이아웃 기법을 보여줍니다.
- **적용**: 소나기 v3.0은 2px 보더를 쓰는 'Wild' 모드와 대비되는 **'Spaced-only'** 모드를 'Thin' 레이어의 서브 옵션으로 고려합니다. 보더가 사라졌을 때 정보가 파편화되지 않게 만드는 MONO의 여백 규칙을 참고합니다.

### 2.3 High-Contrast Accessibility
- **적용**: 흑백(Monochrome) 환경에서도 정보의 중요도가 묻히지 않도록 만드는 **고대비(High-Contrast) 설계**를 소나기의 다크 모드와 고대비 접근성 테마에 반영합니다.
