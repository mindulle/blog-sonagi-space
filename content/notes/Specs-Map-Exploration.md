# 🗺️ Detailed Specs: Map Exploration (US-101, 102)

이 문서는 지도 기반의 탐색 및 장소 상세 확인 기능에 대한 상세 비즈니스 로직과 수락 기준(Acceptance Criteria)을 정의합니다.

---

### 1. [US-101] 내 주변 절약 포인트 탐색 (Main Map)

#### 1.1 위치 기반 POI 렌더링
*   **초기 로딩**: 앱 실행 시 사용자의 GPS 기반 현재 위치를 중심으로 지도를 로드한다.
*   **필터 기본값**: '전체' 카테고리, '5,000원 이하' 혹은 '무료' 시설을 우선순위로 노출한다.
*   **클러스터링**: 줌 레벨에 따라 인접한 마커들을 그룹화하여 숫자로 표시하며, 일정 레벨 이상 확대 시 개별 마커를 노출한다.

#### 1.2 마커 시각화 (Marker Design)
*   **가격 중심 노출**: 마커 아이콘 상단에 해당 장소의 '최저가 메뉴 가격' 혹은 '무료' 텍스트를 배지 형태로 상시 노출한다.
*   **카테고리 구분**: 아이콘 색상 혹은 심볼을 통해 업종(식당, 편의점, 공공시설 등)을 구분한다.

#### 1.3 가격 필터링 (Price Filtering)
*   **방식**: 프리셋 버튼(Preset Buttons) 제공. 사용자가 일일이 입력하거나 슬라이더를 조절하지 않고도 빠르게 원하는 금액대 선택 가능.
*   **필터 프리셋 종류**:
    *   **전체**: 모든 가격대 노출
    *   **무료**: 공공시설, 무료 이벤트 등 가격 정보가 '0'인 POI
    *   **3,000원 이하**: 초저가 간식 및 식사
    *   **5,000원 이하**: 표준 '거지맵' 타겟 식사
    *   **10,000원 이하**: 가성비 식당 한계선
*   **상태 관리**: 한 번에 하나의 필터만 활성화(Radio)되거나, 복수 선택(Checkbox) 가능 여부는 기획 단계에서 확정 필요 (현재는 단일 선택 권장).

### 2. [US-102] 장소 상세 정보 확인 (Shop Detail Overlay)

#### 2.1 오버레이 노출 로직
*   **진입**: 지도상의 마커를 클릭하면 화면 하단에서 슬라이드 업(Slide-up) 형태의 낱장 오버레이(Sheet)가 등장한다.
*   **닫기**: 지도 빈 공간 클릭, 오버레이 아래로 스와이프, 혹은 닫기(X) 버튼 클릭 시 제거한다.

#### 2.2 정보 계층 구조 (Information Hierarchy)
*   **헤더**: 장소명, 카테고리, 현재 위치로부터의 거리(m/km).
*   **핵심 정보 (The "Beggar" Info)**: 
    *   대표 저렴 메뉴 (예: "잔치국수 4,000원")
    *   절약 팁 (예: "현금 결제 시 500원 할인", "물은 셀프 서비스")
*   **추가 정보**: 영업시간, 상세 주소, 전화번호(클릭 시 통화 연결).

### 3. 데이터 요구사항 (Data Requirements)

| 필드명 | 타입 | 설명 | 필수 여부 |
| :--- | :--- | :--- | :--- |
| `poi_id` | UUID | 장소 고유 식별자 | Y |
| `name` | String | 장소 이름 | Y |
| `category` | Enum | 식당, 편의점, 공용공간 등 | Y |
| `latitude` | Float | 위도 | Y |
| `longitude` | Float | 경도 | Y |
| `min_price` | Integer | 최저가 메뉴 가격 | Y |
| `savings_tip` | Text | 사용자에게 도움되는 절약 꿀팁 | N |
