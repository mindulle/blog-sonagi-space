# ☁️ 클라우드컴퓨팅 중간고사 단권화 노트

> 본 문서는 수업 강의 자료(클라우드 아키텍처, 서비스 모델, Docker, 퍼블릭/프라이빗 클라우드)를 기반으로 작성된 중간고사 대비 요약본입니다.

## 📌 1. 클라우드 컴퓨팅 시스템 디자인 및 확장
* **클라우드 컴퓨팅의 정의**: 확장 가능하고 탄력적인 IT 기능이 인터넷을 사용하는 외부 고객들에게 서비스 형태로 제공되는 컴퓨팅 방식.
* **시스템 확장(Scaling) 방식**:
  1. **수직적 확장 (Vertical Scaling / Scale-up, Scale-down)**: 기존 시스템의 장비/부품을 고사양으로 교체하여 성능을 향상 (예: RAM 추가, 고성능 CPU 교체).
  2. **수평적 확장 (Horizontal Scaling / Scale-out, Scale-in)**: 동일한 장비를 여러 대 추가하여 부하를 분산 (예: 웹 서버 대수 증가). 클라우드에서 주로 선호됨.

## 📌 2. 핵심 클라우드 아키텍처 (8가지)
1. **작업 부하 분산 아키텍처 (Workload Distribution)**: 로드 밸런서를 통해 여러 IT 자원 간에 작업 부하를 균등하게 배분.
2. **자원 풀링 아키텍처 (Resource Pooling)**: 물리/가상 서버, 스토리지 등을 풀(Pool)로 구성하여 프로비저닝 단계에서 사용자에게 동적으로 할당.
3. **동적 확장 아키텍처 (Dynamic Scalability)**: 확장 리스너(Listener)를 통해 작업 부하를 모니터링하고 미리 정의된 조건에 따라 가변적으로 용량을 할당(수평/수직 확장).
4. **탄력적 자원 용량 아키텍처 (Elastic Resource Capacity)**: 수요에 즉각적으로 대응하여 자원을 할당하거나 회수 (하이퍼바이저와 상호작용).
5. **서비스 로드 밸런싱 아키텍처 (Service Load Balancing)**: 로드 밸런서가 부하 분산을 수행. 내/외부에 배치될 수 있음.
6. **클라우드 버스팅 아키텍처 (Cloud Bursting)**: 온프레미스 자원의 임계치 도달 시, 퍼블릭 클라우드 환경으로 동적으로 확장(Burst-out)하여 부하를 처리. 상태 관리를 위해 DB 동기화 필요.
7. **탄력적 디스크 프로비저닝 (Elastic Disk Provisioning)**: 실제 사용하는 용량만큼만 스토리지 요금을 지불하도록 하는 '씬 프로비저닝(Thin Provisioning)' 활용.
8. **이중화 스토리지 아키텍처 (Redundant Storage)**: 스토리지 장애에 대비하여 주/보조 클라우드 스토리지로 이중화하고 데이터를 정기적으로 동기화하여 가용성과 신뢰성 확보.

## 📌 3. 주요 기술 및 서비스 모델
* **서비스 모델**:
  * **IaaS (Infrastructure)**: 가상 머신, 스토리지, 네트워크 인프라 제공 (예: AWS EC2).
  * **PaaS (Platform)**: 애플리케이션 개발/실행 플랫폼 제공.
  * **SaaS (Software)**: 완성된 소프트웨어를 웹을 통해 제공.
  * **FaaS (Function)**: 서버 관리 없이 코드(함수) 단위로 실행하고, 실행된 시간만큼만 과금 (Serverless).
* **가상화 기술 (Docker)**:
  * 기존 하이퍼바이저 기반 가상머신(VM)과 달리, OS를 공유하면서 애플리케이션과 라이브러리만 격리하는 **'컨테이너(Container)'** 가상화 도구. 더 가볍고 빠름.
* **SLA (Service Level Agreement)**:
  * 클라우드 사업자(AWS, Azure, GCP 등)와 고객 간에 맺는 서비스 품질 보장 계약 (가용성 퍼센트 등 명시).

## 📌 4. 퍼블릭 및 프라이빗 클라우드 실습 환경
* **퍼블릭 클라우드**: Amazon AWS, Microsoft Azure, GCP 등. 로드 밸런서(LB)와 가상 머신 스케일 셋(VMSS)을 통한 자동 확장 구현.
* **프라이빗 클라우드**: **OpenStack (DevStack)**을 사용하여 자체적인 로컬 클라우드 인프라 구축 및 가상 머신(인스턴스) 운영.
