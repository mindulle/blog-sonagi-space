# 과제 6. Range in Mowkow

**학번: 201524471**

**이름: 박혜성**

### **1. 문제 설명**

표준 입력으로 주어진 두 정수(시작값, 끝값)에 대하여, 두 수 사이의 모든 정수를 포함하는 리스트를 생성하는 프로그램을 작성하는 문제입니다

- **포함 규칙:** 입력받은 양 끝의 정수를 모두 리스트에 포함해야 합니다
- **예외 처리:** 만약 시작값이 끝값보다 크면 빈 리스트(**`공`** 또는 **`()`**)를 반환해야 합니다
- **제약 사항:** 모든 식별자(함수명, 변수명)는 한글로 작성하며, 메모리 효율을 위해 **꼬리 재귀 호출(Tail-Recursive Call)** 방식을 사용해야 합니다.

### **2. 전략**

이번 과제에서는 리스트를 효율적으로 생성하기 위해 **누적자(Accumulator)**를 활용한 꼬리 재귀 전략을 채택하였습니다.

- **보조 함수 정의:** **`(범위생성 시작 끝 누적)`** 함수를 정의하여, 현재까지 생성된 리스트를 **`누적`** 인수에 담아 전달합니다.
- **기저 사례(Base Case):** **`시작`**값이 **`끝`**값보다 커지는 순간, 더 이상 추가할 숫자가 없으므로 지금까지 쌓인 **`누적`** 리스트를 즉시 반환하고 종료합니다.
- **재귀 단계(Recursive Step):** **`끝`**값을 1씩 감소시키며, 현재의 **`끝`**값을 **`누적`** 리스트의 맨 앞에 추가(**`짝`**)합니다. 이 연산이 함수의 마지막 작업(Tail Position)이 되도록 설계하여 꼬리 재귀 최적화가 가능하게 하였습니다.
- **메인 로직:** **`읽기`** 함수로 두 값을 입력받고, 초기 누적값으로 빈 리스트 **`()`**를 전달하여 함수를 호출한 뒤 결과를 **`쓰기`**로 출력합니다.

### **3. 언어의 기능**

프로그램 구현에 활용된 주요 기능은 다음과 같습니다.

- **꼬리 호출 최적화(Tail-call Optimization):** 스택 프레임을 재사용하여 깊은 재귀에서도 스택 오버플로 없이 안전하게 리스트를 생성할 수 있습니다.
- **한글 식별자 지원:** **`범위생성`**, **`시작`**, **`끝`**, **`누적`** 등 우리말 변수명을 사용하여 로직의 가독성을 높였습니다.
- **리스트 기본 함수:** 리스트의 생성자 역할을 하는 **`짝(cons)`**을 통해 동적으로 정수 집합을 구축할 수 있었습니다.

### **4. 토의**

본 과제에서 구현한 꼬리 재귀 방식과 대비되는 **일반 재귀(Standard Recursion)** 방식의 코드를 비교 분석하였습니다.

**[대안 코드: 일반 재귀 방식]**

```
(정의 (레인지 시작 끝)
  (만약 (> 시작 끝)
    ()
    (짝 시작 (레인지 (+ 시작 1) 끝))))
```

- **비교 분석:** 일반 재귀 방식은 **`(짝 시작 ...)`** 연산을 수행하기 위해 재귀 호출 결과가 돌아올 때까지 기다려야 하므로, 호출 깊이만큼 스택 메모리를 소모합니다. 반면, 본인이 작성한 꼬리 재귀 방식(**`범위생성`**)은 상태를 인수로 넘기기 때문에 추가적인 스택 소모 없이 루프(loop)와 동일한 효율을 보입니다. 또한, 본인의 코드는 큰 숫자부터 거꾸로 **`짝`**을 지어 리스트를 구성함으로써 최종적으로 오름차순 리스트를 얻는 영리한 방식을 취하고 있습니다.

### **5. 결론**

함수형 언어인 머꼬를 통해 반복문 없이 오직 재귀만으로 복잡한 자료구조(리스트)를 생성하는 원리를 깊이 이해할 수 있었습니다. 특히 꼬리 재귀를 위해 누적자를 사용하는 기법은 기존 명령형 언어의 사고방식에서 벗어나 '상태의 전이'라는 관점에서 문제를 바라보게 해주었습니다. 한글로 코딩하며 논리 구조를 우리말로 세우는 과정이 매우 흥미로웠으며, 괄호의 개수를 맞추는 작업은 여전히 까다로웠으나 S-식(S-Expression)의 단순함이 주는 매력을 느꼈습니다.
