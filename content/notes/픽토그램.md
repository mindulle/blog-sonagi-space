# 픽토그램 (Pictogram)

**픽토그램(Pictogram)**은 사물, 시설, 형태, 개념 등을 대중들이 언어의 장벽 없이 직관적으로 이해할 수 있도록 **가장 단순하게 압축한 그림 문자(Picture + Telegram)**입니다. 

현대 UI/UX 디자인에서는 주로 **아이콘(Icon)**으로 불리며, 복잡한 텍스트를 대체하여 화면의 인지 속도를 높이는 데 사용됩니다.

## 1. 픽토그램 디자인의 3대 원칙 (ISO 국제 표준 기반)

### 1.1 극단적인 단순화 (Simplicity)
- 사실적인 묘사(일러스트)가 아니라 기호학적 접근이 필요합니다. 올리버 번(Oliver Byrne)이 기하학으로 수학을 증명했듯, **원, 삼각형, 사각형, 직선** 등 가장 기본적인 도형의 결합만으로 대상을 표현해야 합니다.
- 불필요한 장식, 원근감, 그림자를 모두 제거합니다. (미니멀리즘)

### 1.2 즉각적인 인지성 (Recognizability)
- 사용자가 픽토그램을 보고 1초 이내에 의미를 파악해야 합니다. 
- 돋보기(검색), 톱니바퀴(설정), 집(홈)처럼 이미 전 세계적으로 통용되는 **멘탈 모델(Mental Model)**이 있다면, 굳이 창의적으로 새로 만들기보다는 표준을 따르는 것이 UX적으로 훨씬 훌륭한 디자인입니다.

### 1.3 시스템적 통일성 (Consistency)
- 하나의 앱이나 서비스 안에서 쓰이는 픽토그램 세트는 반드시 **동일한 뼈대(Grid)** 위에서 제작되어야 합니다.
- 선의 굵기(Stroke weight), 모서리의 둥글기(Border radius), 채워진 면(Fill)과 선(Line)의 사용 여부가 세트 전체에서 완벽하게 통일되어야 합니다.

## 2. 픽토그램 작도의 실무 (Icon Grid System)

피그마(Figma)나 일러스트레이터에서 픽토그램을 그릴 때는 프리핸드로 그리지 않고 반드시 **아이콘 그리드(Icon Grid)**를 베이스로 씁니다.

1. **24x24 또는 32x32 베이스**: 보통 가로세로 24px의 정사각형을 도화지로 삼습니다.
2. **라이브 에어리어 (Live Area)**: 24px 도화지 안에서, 양끝 2px씩은 여백으로 비워두고 실제 그림은 20x20 픽셀 안쪽에만 꽉 차게 그립니다. (모든 아이콘의 시각적 볼륨감을 일치시키기 위함)
3. **키셰이프 (Keylines)**: 원형 아이콘, 가로로 긴 직사각형, 세로로 긴 직사각형이 서로 비슷한 크기로 보이도록 안내선을 긋고 그 안에서 조립합니다.
