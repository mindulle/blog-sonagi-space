# Cross Join

A Cross Join produces a result set that is the number of rows in the first table multiplied by the number of rows in the second table. If a WHERE clause is used in conjunction with a CROSS JOIN, it functions like an INNER JOIN. However, using an INNER JOIN is generally preferred to using a CROSS JOI
N with a WHERE clause for readability and performance reasons. It essentially creates all possible combinations of rows from the tables involved.
